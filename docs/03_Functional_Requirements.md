# 3. Функциональные требования

## 3.1 Общее описание

В данном разделе описываются функциональные требования к backend-системе **RoadVision AI**.

Все требования сгруппированы по функциональным блокам и имеют уникальные идентификаторы.

**Формат идентификатора:** `FR-[КАТЕГОРИЯ]-[НОМЕР]`

**Категории:**
- `AUTH` — аутентификация и управление пользователями
- `UPLOAD` — загрузка медиафайлов
- `PROCESS` — обработка и анализ
- `DEFECT` — управление дефектами
- `MAP` — карта и визуализация
- `REALTIME` — работа с камерами и потоками
- `REPORT` — отчёты
- `SYSTEM` — системные функции

---

## 3.2 Аутентификация и управление пользователями (AUTH)

| ID | Требование | Роль | Приоритет |
|----|------------|------|-----------|
| **FR-AUTH-01** | Система должна поддерживать регистрацию пользователей по email и паролю | Все | High |
| **FR-AUTH-02** | Система должна поддерживать вход в систему (login) | Все | High |
| **FR-AUTH-03** | Система должна поддерживать выход из системы (logout) | Все | High |
| **FR-AUTH-04** | Система должна поддерживать сброс пароля | Все | Medium |
| **FR-AUTH-05** | Система должна поддерживать роли: user, operator, admin | Все | High |
| **FR-AUTH-06** | Только администратор может изменять роли пользователей | Admin | High |
| **FR-AUTH-07** | Система должна использовать JWT для аутентификации | Все | High |
| **FR-AUTH-08** | Система должна поддерживать refresh/access токены | Все | Medium |
