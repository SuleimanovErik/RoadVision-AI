# 1. Концепция проекта

## 1.1 Название проекта

**RoadVision AI** — система автоматического обнаружения дефектов дорожного покрытия с использованием технологий компьютерного зрения.

---

## 1.2 Актуальность проекта

Состояние дорожной инфраструктуры является одной из ключевых проблем в современных городах. Несвоевременное выявление дефектов дорожного покрытия (ям, трещин, повреждений разметки) приводит к увеличению аварийности, росту затрат на ремонт и снижению качества городской среды.

Традиционные методы мониторинга дорог, такие как визуальный осмотр и выездные инспекции, обладают рядом недостатков:
- высокая трудоёмкость
- субъективность оценки
- отсутствие возможности непрерывного мониторинга

В связи с этим возникает необходимость разработки автоматизированных решений, способных в реальном времени анализировать состояние дорожного покрытия.

Система **RoadVision AI** направлена на решение данной проблемы за счёт применения методов компьютерного зрения и обработки видеоданных.

---

## 1.3 Цель проекта

Целью проекта является разработка backend-системы на базе **Django REST Framework**, обеспечивающей автоматическое обнаружение дефектов дорожного покрытия и управление данными мониторинга.

Система должна обеспечивать:
- приём изображений и видео от пользователей
- автоматический анализ данных с использованием моделей компьютерного зрения (YOLO)
- сохранение информации о дефектах с географическими координатами
- поддержку мониторинга в режиме реального времени через RTSP-потоки
- формирование аналитических отчётов

---

## 1.4 Задачи проекта

Для достижения поставленной цели необходимо решить следующие задачи:

1. Разработать REST API для загрузки изображений и видео с привязкой к GPS-координатам.
2. Интегрировать модель компьютерного зрения (YOLO) для обнаружения дефектов:
   - POTHOLE (ямы)
   - CRACK (трещины)
   - ALLIGATOR_CRACK (крокодиловые трещины)
   - ROAD_MARKING (повреждение разметки)
3. Реализовать два режима обработки данных:
   - **Offline-режим** (анализ загруженных медиафайлов)
   - **Real-time режим** (анализ видеопотока)
4. Разработать механизм кластеризации дефектов для устранения дубликатов.
5. Реализовать систему модерации дефектов (подтверждение/редактирование).
6. Обеспечить генерацию отчётов в формате PDF.
7. Спроектировать масштабируемую архитектуру с использованием:
   - Celery (фоновые задачи)
   - Redis (очереди и кеширование)
   - Django Channels (WebSocket)

---

## 1.5 Роли пользователей

В системе предусмотрено три уровня доступа:

- **Пользователь (User)**  
  - загрузка фото и видео  
  - просмотр карты дефектов  

- **Оператор (Operator)**  
  - управление RTSP-камерами (добавление, запуск, остановка)  
  - просмотр видеопотока в реальном времени  

- **Администратор (Admin)**  
  - все возможности User и Operator  
  - подтверждение и редактирование дефектов  
  - управление пользователями  
  - удаление камер  
  - генерация отчётов  

---

## 1.6 Основные сценарии работы системы

### Offline-режим

1. Пользователь загружает изображение или видео с координатами.
2. Данные отправляются на backend через API.
3. Запускается обработка с использованием AI-модели.
4. При обнаружении дефектов информация сохраняется в базе данных.
5. Дефекты отображаются на карте.

---

### Real-time режим

1. RTSP-камера передаёт видеопоток.
2. Система обрабатывает поток в реальном времени.
3. При обнаружении дефекта:
   - данные сохраняются в базе
   - информация отправляется клиенту через WebSocket
4. Дефекты отображаются на карте.

---

## 1.7 Ключевые особенности системы

- Автоматическое обнаружение дефектов с использованием AI
- Поддержка offline и real-time режимов
- Сохранение только релевантных данных (с дефектами)
- Геопривязка всех обнаружений
- Кластеризация дефектов (устранение дубликатов)
- Масштабируемая архитектура
- Разграничение прав доступа (RBAC)
