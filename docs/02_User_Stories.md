# 2. Пользовательские требования (User Stories)

## 2.1 Роли пользователей

В системе определены следующие роли:

- **Пользователь (User)**  
  Может загружать фотографии и видео дорог, а также просматривать карту дефектов.

- **Оператор (Operator)**  
  Отвечает за управление камерами видеонаблюдения и мониторинг видеопотоков в реальном времени.

- **Администратор (Admin)**  
  Имеет полный доступ ко всем функциям системы:
  - все возможности пользователя
  - все возможности оператора
  - управление дефектами
  - управление пользователями
  - генерация отчётов

---

## 2.2 Пользовательские сценарии

### 2.2.1 Аутентификация и управление аккаунтом

- **US-01**: Пользователь хочет зарегистрироваться в системе, чтобы получить доступ к функционалу.
- **US-02**: Пользователь хочет войти в систему, чтобы загружать медиафайлы и просматривать дефекты.
- **US-03**: Пользователь хочет восстановить пароль, чтобы получить доступ к аккаунту при его утере.
- **US-04**: Администратор хочет управлять ролями пользователей, чтобы назначать права оператора или администратора.

---

### 2.2.2 Загрузка медиафайлов (Пользователь)

- **US-05**: Пользователь хочет загружать фотографии дорог с GPS-координатами, чтобы система могла обнаруживать дефекты.
- **US-06**: Пользователь хочет загружать видео дорог с GPS-координатами, чтобы система могла анализировать несколько кадров.
- **US-07**: Пользователь хочет видеть статус обработки загруженных файлов, чтобы понимать текущий этап анализа.
- **US-08**: Пользователь хочет просматривать историю своих загрузок, чтобы отслеживать отправленные данные.

---

### 2.2.3 Просмотр результатов (Пользователь)

- **US-09**: Пользователь хочет просматривать дефекты на карте, чтобы видеть состояние дорог.
- **US-10**: Пользователь хочет фильтровать дефекты по типу и дате, чтобы быстро находить нужную информацию.
- **US-11**: Пользователь хочет просматривать подробную информацию о дефекте (тип, координаты, изображение, уверенность модели), чтобы понимать результат анализа.

---

### 2.2.4 Управление камерами и real-time (Оператор)

- **US-12**: Оператор хочет добавлять камеры видеонаблюдения, указывая RTSP-адрес, чтобы начать мониторинг.
- **US-13**: Оператор хочет запускать и останавливать видеопотоки, чтобы управлять обработкой данных.
- **US-14**: Оператор хочет просматривать видео в реальном времени с отображением обнаруженных дефектов, чтобы контролировать ситуацию на дорогах.
- **US-15**: Оператор хочет просматривать список активных и неактивных камер, чтобы отслеживать их состояние.

---

### 2.2.5 Управление дефектами (Администратор)

- **US-16**: Администратор хочет просматривать все обнаруженные дефекты, чтобы контролировать систему.
- **US-17**: Администратор хочет подтверждать корректные дефекты, чтобы повысить достоверность данных.
- **US-18**: Администратор хочет отклонять ложные срабатывания, чтобы исключить ошибки модели.
- **US-19**: Администратор хочет редактировать информацию о дефекте (тип, серьёзность, комментарий), чтобы корректировать результаты ИИ.
- **US-20**: Администратор хочет удалять некорректные дефекты, чтобы поддерживать качество данных.

---

### 2.2.6 Управление системой (Администратор)

- **US-21**: Администратор хочет управлять пользователями (создавать, изменять, удалять), чтобы контролировать доступ к системе.
- **US-22**: Администратор хочет удалять камеры видеонаблюдения, чтобы поддерживать актуальность системы.

---

### 2.2.7 Отчёты (Администратор)

- **US-23**: Администратор хочет генерировать PDF-отчёты по дефектам, чтобы использовать их в официальной документации.
- **US-24**: Администратор хочет фильтровать отчёты по дате, типу дефекта и локации, чтобы получать релевантную информацию.

---

### 2.2.8 Системные сценарии

- **US-25**: Система должна автоматически анализировать загруженные фото и видео с помощью модели YOLO.
- **US-26**: Система должна обрабатывать видеопотоки с камер в реальном времени.
- **US-27**: Система должна предотвращать дублирование дефектов путём кластеризации близко расположенных точек.
- **US-28**: Система должна удалять медиафайлы, если дефекты не обнаружены.

---

## 2.3 Критерии приемки (примеры)

### US-05 — Загрузка фотографии

**Критерии:**
- Пользователь должен быть авторизован
- Запрос содержит файл изображения и GPS-координаты
- Поддерживаемые форматы: JPG, PNG
- После обработки:
  - если дефекты найдены → данные сохраняются
  - если дефекты не найдены → файл удаляется

---

### US-14 — Просмотр видео в реальном времени

**Критерии:**
- Доступ только у оператора или администратора
- Видео отображается в реальном времени
- На изображении отображаются bounding boxes
- Задержка не превышает 3 секунд
