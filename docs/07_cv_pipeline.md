# 7. Алгоритмы и обработка данных (Computer Vision Pipeline)

## 7.1 Общее описание

Модуль компьютерного зрения (Computer Vision, CV) является ключевым компонентом системы **RoadVision AI** и отвечает за автоматическое обнаружение дорожных дефектов на изображениях, видео и в потоках реального времени.

Основная задача модуля — анализ визуальных данных и выделение дефектов дорожного покрытия с высокой точностью.

---

## 7.2 Используемые технологии

В системе используются следующие технологии:

- **Python**
- **OpenCV** — обработка изображений и видео
- **YOLO (You Only Look Once)** — детекция объектов
- **NumPy** — работа с массивами
- **Torch / Ultralytics YOLO** — инференс модели

---

## 7.3 Общий pipeline обработки

Обработка данных в системе реализована в виде последовательного pipeline:

1. Получение данных (изображение / видео / поток)
2. Предобработка
3. Запуск модели CV
4. Постобработка результатов
5. Фильтрация и сохранение

---

## 7.4 Обработка изображений

### Этапы:

1. Пользователь загружает изображение
2. Backend сохраняет файл во временное хранилище
3. Изображение передаётся в CV-модуль
4. Выполняется инференс модели YOLO
5. Из результата извлекаются:
   - bounding box
   - тип дефекта
   - confidence
6. Если дефекты обнаружены:
   - создаётся запись в БД
   - изображение сохраняется
7. Если нет:
   - изображение удаляется

---

## 7.5 Обработка видео

Видео обрабатывается покадрово.

### Этапы:

1. Видео разбивается на кадры (frames)
2. Каждый кадр передаётся в модель YOLO
3. При обнаружении дефекта:
   - фиксируется timestamp
   - создаётся запись Defect
4. Результаты агрегируются

📌 Для оптимизации:
- анализируется не каждый кадр (например, 1 из N)
- используется ограничение FPS

---

## 7.6 Обработка RTSP-потока (real-time)

### Этапы:

1. Оператор запускает камеру
2. Устанавливается соединение с RTSP-потоком
3. Поток разбивается на кадры
4. Кадры передаются в CV
5. При обнаружении дефекта:
   - создаётся запись Defect
   - привязка к StreamSession

---

## 7.7 Алгоритм детекции (YOLO)

Модель YOLO выполняет детекцию объектов за один проход по изображению.

### Основные принципы:

- изображение делится на сетку
- для каждой ячейки предсказываются bounding box
- вычисляется вероятность объекта
- применяется Non-Max Suppression (NMS)

---

## 7.8 Постобработка результатов

После получения предсказаний выполняется:

- фильтрация по `confidence` (например > 0.5)
- преобразование координат bbox
- определение типа дефекта
- расчёт серьёзности (`severity`)

---

## 7.9 Кластеризация дефектов

Для предотвращения дублирования используется алгоритм кластеризации:

1. Для каждого нового дефекта ищутся ближайшие
2. Если расстояние < заданного радиуса:
   - дефект добавляется в существующий кластер
3. Иначе создаётся новый кластер

📌 Используется геодезическое расстояние (по координатам)

---

## 7.10 Производительность и оптимизация

Для повышения эффективности используются:

- уменьшение разрешения изображения
- обработка не каждого кадра
- асинхронная обработка (в перспективе)
- ограничение частоты обработки (FPS)

---

## 7.11 Особенности реализации

- единый pipeline для всех источников данных
- строгая фильтрация (сохраняются только дефекты)
- поддержка real-time обработки
- масштабируемость (возможен вынос CV в отдельный сервис)

---

## 7.12 Вывод

Реализованный pipeline обеспечивает эффективную и масштабируемую обработку визуальных данных, позволяя системе автоматически обнаруживать дорожные дефекты в различных сценариях: загрузка файлов и потоковое видео.
